# Competitive teardown — sources (2026-07-08, for redesign brief v2)

Fresh per-project research per CLAUDE.md §2 (CDO) / §4.9. Patterns borrowed, never pixels.

## Granola — invisible design
- One-tap capture, zero config before value; bot-free keeps focus on the conversation.
- User's own notes render black; AI-generated text grey — ownership is visually legible.
- "Second brain that sits quietly next to you" — supportive, not controlling.
- Sources: [Efficient App review](https://efficient.app/apps/granola) · [tl;dv review](https://tldv.io/blog/granola-review/) · [Wonder Tools guide](https://wondertools.substack.com/p/granolaguide) · [BlueDot review](https://www.bluedothq.com/blog/granola-review)

## Wispr Flow — out-of-the-box polish
- Works with no configuration; onboarding polish is itself a differentiator ("by far the best UI").
- Output auto-adapts to destination app — users never visit settings.
- Safety nets (auto-clipboard copy) build trust invisibly.
- Sources: [Efficient App review](https://efficient.app/apps/wispr-flow) · [Spokenly review](https://spokenly.app/blog/wispr-flow-review) · [tl;dv review](https://tldv.io/blog/wisprflow/)

## Linear — craft = noise reduction + speed
- Redesign explicitly reduced visual noise and raised hierarchy/density of navigation.
- Command palette as universal action/feature index; performance treated as a design feature.
- Source: [How we redesigned the Linear UI](https://linear.app/now/how-we-redesigned-the-linear-ui)

## Onboarding research (activation science)
- Aha ≤5 min → ~40% higher 30-day retention vs 15+ min; TTFV target <15 min, ideally minutes.
- 30–40% of onboarding steps are typically removable; audit each step against "does this move the user closer to value?"
- Progressive disclosure beats feature dumps; defer config to contextual moments.
- Sources: [Appcues onboarding strategies](https://www.appcues.com/blog/8-user-onboarding-strategies) · [Digital Applied TTV framework 2026](https://www.digitalapplied.com/blog/customer-onboarding-time-to-value-2026-saas-metrics-framework) · [Userpilot onboarding 2026](https://userpilot.com/blog/user-onboarding/) · [UserGuiding best practices](https://userguiding.com/blog/user-onboarding-best-practices)

## Best parts to take
1. Record = one unmistakable action (Granola / Voice Memos) → rail CTA, not nav row.
2. Ownership legibility (Granola black-vs-grey) → You/Everyone-else + user-notes vs AI-notes contrast.
3. Zero-config value (Wispr) → onboarding v2 defers keys/calendar/enrollment to context.
4. Command palette as discoverability index (Linear) → Ctrl+K lists every feature by name.
5. Aha-first onboarding (research) → 20-second test recording with live transcript payoff.
